## Abstract

Formal education pushes technical skills through set curricula, but I've learned serious tech skills outside classrooms by jumping into real problems. This auto-ethnographic case study examines my journey as a student builder (2021-2025), analyzing projects like [MediLink Healthcare Assistant](https://github.com/rupinajay/MediLink-Healthcare-Assistant), Git commits, and notes to map skill acquisition without syllabi. I identified a "Need–Build–Validate" loop driven by problem triggers, validated by functional prototypes. With AI tools accelerating learning in 2025-2026, this suggests new approaches for education and credentialing.[1]

## 1. Introduction

Traditional curricula teach tech skills linearly: HTML → CSS → JavaScript → React. But I learned React by building MediLink's dynamic UI because the project demanded it. Schools miss this—problem-driven learning where online docs, GitHub, and AI replace teachers.

**Research Question:** How do independent student builders construct technical skills without formal curriculum?

## 2. Related Work

**Self-Directed Learning (SDL):** Knowles (1975) identified motivation → planning → reflection cycles.[2] Zimmerman (2002) showed SDL develops through assessment → monitoring → adjustment.[3]

**Problem-Based Learning (PBL):** Hmelo-Silver (2004) found PBL yields 3.98 point gains vs 1.02 for lectures.[4]

**Recent Evidence:** 2025 studies confirm SDL remains crucial post-COVID despite online learning challenges.[5][6]

**Portfolios vs Tests:** Companies using project evaluation see 40% less turnover, 50% better talent identification.[7]

**Gap:** Most research examines MOOCs/bootcamps. None deeply analyze fully independent builders.

## 5. Discussion

### 5.1 Theoretical Synthesis

Combines SDL metacognition[3], PBL triggers[4], experiential learning cycles. Curriculum-independent learning stays current; curricula lag 2-3 years behind tech.

**Trade-offs:**
Depth gained: React/Node/Docker → production-ready
Depth sacrificed: Advanced algorithms → rarely needed
Meta-skill gained: Rapid domain acquisition


### 5.2 Hybrid Education Design

Problem-First: Start with MediLink-style challenges

Portfolio Assessment: GitHub over exams

AI Literacy: Prompt engineering curriculum

OSS Integration: Required community contributions

### 5.3 Credentialing Evolution

Portfolio Credentials → GitHub Verified
Micro-Certs → "React Mastery", "Docker Deployment"
Prior Learning → CAEL-style assessment

## 6. Conclusion

**Need–Build–Validate** explains curriculum-free expertise. Perfect for rapid tech evolution, unequal education access.

**For Institutions:** Hybridize with real problems + public portfolios  
**For Builders:** Make loops intentional via trigger tracking  
**Open Question:** Scalable for average-motivation students?

**Future Work:** Hardware domains, AI skill erosion tracking (2026+)
